# Data Collection

What we can pull from HealthKit on an iPhone (no Apple Watch required). Each item maps to an `HKQuantityType` / `HKCategoryType` and requires explicit user permission.

## From iPhone sensors directly

Generated by the M-series motion coprocessor — available on essentially every modern iPhone:

- Step count
- Distance walked / run
- Flights climbed
- Walking speed, step length, asymmetry, double-support %
- Stair ascent / descent speed
- Active energy burned (estimated)

## From Apple Watch (if the user has one — flows into the same store)

- Heart rate, resting HR, walking HR average
- Heart rate variability (HRV)
- Blood oxygen (SpO₂)
- ECG
- Workouts (`HKWorkout` with type, duration, energy, route)
- Sleep stages
- VO₂ max
- Stand hours, exercise minutes, move calories

## From third-party apps / connected devices

Whatever the user has granted to Health — e.g.:

- Body mass, body fat %, lean mass (smart scales)
- Blood pressure (BP monitors)
- Blood glucose (CGMs)
- Nutrition: calories, macros, water (MyFitnessPal etc.)
- Workouts from Strava, Nike Run Club, etc.

## Manual entries

Anything the user typed into the Health app (weight, mood, symptoms, cycle tracking, etc.).

## Practical notes

- Permissions are **per-type** and granular — the user can grant steps but deny heart rate.
- We cannot detect *denial* — a denied type just returns no data. Design UX to handle empty series.
- Background delivery (`enableBackgroundDelivery`) lets us wake the app on new samples for selected types.
- For incremental sync, use `HKAnchoredObjectQuery` and persist the anchor per type.
